# Company Start Master Plan — Registration, Trust, GTM, and Operating System

**Status:** Draft deployment playbook; founder/adviser review required
**Last updated:** 2026-04-17
**Purpose:** One high-level map for starting and operating the company so apps, micro-SaaS, and consulting services can be launched with trust, commercial readiness, and risk controls.


## Company mandate alignment

Interpret this document through `00-company-mandate-and-alignment.md`: the company is an AI-first founder-led software company using senior software engineering and ecommerce/logistics expertise to build B2B/B2C apps, SaaS/micro-SaaS, consulting, R&D, OSS/content authority, and multiple revenue streams. Dutch ZZP, TapTap, SpatialSense, and SoloOS are lanes, not the entire company identity.

> This is an execution playbook, not legal/tax/accounting/immigration advice. For every public or paid step, check `09-claims-compliance-register.md`, `15-liability-and-risk-architecture.md`, and qualified advisers where marked.

## 1. The operating thesis

The company should start as a **serious solo-founder operating company**, not a random portfolio of handles, apps, and services.

```text
Founder credibility
└── Company trust spine
    ├── official site + proof hub
    ├── registration / legal / tax / privacy foundations
    ├── product/service portfolio logic
    ├── market research and customer-language system
    ├── launch and GTM system
    ├── payment/commercial readiness
    └── liability/risk controls
```

## 2. Stage-gated roadmap

| Stage | Theme | Goal | Must-have artifacts | Gate to next stage |
|---|---|---|---|---|
| 0 | Pre-registration clarity | Know what will be registered, why, and under which name | name decision, entity path, domain plan, risk map, adviser list | no unresolved red risk for registration |
| 1 | Registration and trust spine | Be legally/contactably real | KVK/VAT status verified, email/domain, site, privacy/contact/support, registers | official site can be shared safely |
| 2 | Audience and offer validation | Find first paid wedge | ICP interviews, research board, first CTA, waitlist/interview flow, proof assets | repeated pain + commitment signal |
| 3 | First commercial path | Accept first money safely | app-store/MoR/Stripe/service SOW path, accounting, terms, support, refund/cancel flow | test transaction or signed SOW works |
| 4 | Launch | Publicly announce and learn | launch pack, demo, FAQs, support rota, analytics, reply plan | launch learning captured |
| 5 | Operating cadence | Make it repeatable | weekly review, monthly proof report, access/claims/privacy review, KPI dashboard | repeatable proof + revenue/support signal |
| 6 | Scale/select | Expand only proven lanes | dedicated product identity, advanced analytics, paid channels, partners | channel ROI and support capacity proven |

## 3. Company registration and setup sequence

### Pre-registration decisions

- [ ] Confirm founder work authorization / employer/NOC / immigration constraints.
- [ ] Decide legal form: eenmanszaak now vs BV later trigger.
- [ ] Confirm legal/trade name and public brand candidates.
- [ ] Check domain and handle availability.
- [ ] Check KVK trade-name and BOIP/trademark risk.
- [ ] Define activities to register.
- [ ] Define first revenue lanes: apps, web SaaS, consulting, templates.
- [ ] Identify red/prohibited liability projects.
- [ ] Pick accountant/tax adviser shortlist.
- [ ] Pick legal/privacy adviser shortlist.
- [ ] Pick insurance broker/adviser shortlist.

### Registration and admin

- [ ] Register with KVK when timing is approved.
- [ ] Verify VAT/BTW status from Belastingdienst.
- [ ] Decide KOR/VAT strategy with accountant.
- [ ] Set up business bank account if appropriate.
- [ ] Set up accounting software.
- [ ] Create invoice numbering and admin folders.
- [ ] Set up time tracking if WBSO/urencriterium may matter.
- [ ] Set up eHerkenning if RVO/official portals require it.
- [ ] Store official documents in secure business vault.

### Digital trust spine

- [ ] Buy canonical domain(s).
- [ ] Configure business email.
- [ ] Configure SPF/DKIM/DMARC.
- [ ] Create official site/holding page.
- [ ] Add contact/support/privacy/security routes.
- [ ] Create official account list.
- [ ] Create account register.
- [ ] Create processing register.
- [ ] Create claims register.
- [ ] Create proof-permission register.
- [ ] Create AI tools register.

## 4. First 90-day company operating plan

### Days 0–14 — foundation

Deliver:

- domain and email;
- company site skeleton;
- founder LinkedIn draft;
- company LinkedIn draft;
- account/security register;
- claims and processing registers;
- payment path shortlist;
- prohibited-project list;
- first proof library structure.

### Days 15–30 — public trust shell

Deliver:

- founder LinkedIn update;
- company LinkedIn Page;
- GitHub org;
- YouTube Brand Channel;
- Product Hunt founder profile;
- Reddit listening setup;
- first founder intro post;
- first company proof post;
- first 10 reusable proof/content assets.

### Days 31–60 — validation wedge

Deliver:

- choose first public wedge;
- publish landing page or proof page;
- launch one CTA: checklist, waitlist, discovery call, demo, or service inquiry;
- conduct 10–20 interviews or equivalent high-quality conversations;
- synthesize objections and buying triggers;
- choose paid path candidate.

### Days 61–90 — commercial readiness

Deliver one of:

- app-store readiness checklist for TapTap path;
- MoR vs Stripe decision for web SaaS path;
- SOW/contract/insurance review for consulting path.

Also deliver:

- launch pack;
- support flow;
- refund/cancel policy where relevant;
- KPI dashboard;
- monthly proof report.

## 5. Best-thinking framework stack

Use these frameworks as lenses, not as dogma.

| Problem | Framework / author lens | How to use it here |
|---|---|---|
| Company strategy | Richard Rumelt — diagnosis / guiding policy / coherent action | write one honest diagnosis, one guiding policy, 3–5 coherent actions |
| Strategic choice | Lafley & Martin — where to play / how to win | choose first market wedge and why this company can win there |
| Customer discovery | Rob Fitzpatrick — The Mom Test | interview for past behavior, not compliments |
| Startup learning | Eric Ries — build / measure / learn | define smallest paid/behavioral test |
| Positioning | April Dunford — competitive alternative and differentiated value | define what users would do without you and why you are better for a narrow segment |
| Beachhead | Geoffrey Moore — beachhead segment | win one small segment before expanding |
| Jobs-to-be-done | Christensen / Klement | understand progress the customer is trying to make |
| GTM channels | Gabriel Weinberg / Justin Mares — Traction | test one or two channels at a time |
| Product risk | Marty Cagan — value, usability, feasibility, viability | check all four risks before building |
| Execution scope | Shape Up / Basecamp | shape small bets with appetite and boundaries |
| Marketing clarity | StoryBrand / direct-response principles | make homepage clear: who, pain, outcome, CTA |
| Influence ethics | Cialdini | use proof, authority, consistency ethically; avoid manipulation |
| Category narrative | category design / PMM practice | define the old way, new way, stakes, proof |
| Operations | Drucker / systems thinking | make weekly operating cadence visible and measurable |
| Risk | enterprise risk management basics | identify, avoid, transfer, reduce, accept |

## 7. Default “do not do yet” list

- Do not create separate social accounts for every product idea.
- Do not sell legal/tax/accounting/compliance advice.
- Do not accept high-liability consulting work under vague scope.
- Do not launch paid web checkout before payment/tax/support/privacy path is clear.
- Do not launch app subscriptions before restore/cancel/refund/privacy tests pass.
- Do not run ads/pixels before cookie/privacy review.
- Do not claim GDPR/AI Act/EAA/WBSO/VAT compliance unless verified.
- Do not let AI post, DM, comment, or support customers autonomously.

## 8. Operating cadence

### Weekly

- founder post;
- company proof update;
- customer-language synthesis;
- product/service decision update;
- support/privacy/claims check for new public assets.

### Monthly

- access review;
- claims register review;
- processing register review;
- proof roundup;
- financial/admin reconciliation;
- product/account/channel prune decision.

### Quarterly

- entity/liability review;
- payment/tax architecture review;
- positioning/naming review;
- product portfolio review;
- insurance/adviser review.
