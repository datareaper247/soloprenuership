# Company Registration and Admin Checklist

**Status:** Draft; verify with KVK/Belastingdienst/IND/accountant before action
**Last updated:** 2026-04-17
**Purpose:** Practical checklist for registering and operating the Dutch eenmanszaak/company foundation before paid apps, micro-SaaS, or services.


## Company mandate alignment

Interpret this document through `00-company-mandate-and-alignment.md`: the company is an AI-first founder-led software company using senior software engineering and ecommerce/logistics expertise to build B2B/B2C apps, SaaS/micro-SaaS, consulting, R&D, OSS/content authority, and multiple revenue streams. Dutch ZZP, TapTap, SpatialSense, and SoloOS are lanes, not the entire company identity.

## 1. Pre-registration gate

- [ ] Confirm founder work authorization / residence permit / employer NOC status.
- [ ] Confirm whether registration timing conflicts with employment/visa/tax constraints.
- [ ] Choose initial legal form: eenmanszaak now, BV later trigger.
- [ ] Decide public naming path: `Eenmanszaak`, `Eenmanzak`, or distinctive trade/product name.
- [ ] Check KVK trade-name availability.
- [ ] Check BOIP/trademark risk if a distinctive brand is chosen.
- [ ] Check domain/handle availability.
- [ ] Choose registered activities.
- [ ] Define first likely revenue types:
  - app-store revenue;
  - web micro-SaaS;
  - consulting/services;
  - digital products/templates.
- [ ] Read prohibited-project checklist in `15-liability-and-risk-architecture.md`.

## 2. KVK / Belastingdienst / official admin

- [ ] KVK appointment / registration completed.
- [ ] KVK number received.
- [ ] Trade name(s) recorded.
- [ ] Activities recorded accurately.
- [ ] Public address/privacy choice reviewed.
- [ ] Belastingdienst VAT/BTW status received.
- [ ] VAT ID / OB number stored securely.
- [ ] KOR decision made with accountant if relevant.
- [ ] eHerkenning needed? If yes, apply early.
- [ ] WBSO timing reviewed before any qualifying R&D starts.

## 3. Banking and accounting

- [ ] Business bank account selected or separate account strategy documented.
- [ ] Accounting package selected.
- [ ] Accountant/tax adviser selected or shortlist created.
- [ ] Invoice template created.
- [ ] Invoice numbering scheme created.
- [ ] Revenue categories created:
  - App Store / Google Play payouts;
  - MoR payouts;
  - Stripe/direct payments;
  - consulting invoices;
  - affiliate/creator revenue if any.
- [ ] Expense categories created.
- [ ] Receipt storage process created.
- [ ] Monthly reconciliation routine scheduled.
- [ ] 7-year/admin retention requirements reviewed with accountant.

## 4. Insurance and liability

- [ ] Business liability insurance discussed.
- [ ] Professional liability insurance discussed.
- [ ] Cyber insurance discussed if processing client/user data.
- [ ] Legal assistance insurance considered.
- [ ] US/Canada/worldwide exclusions checked.
- [ ] Software/advice/AI exclusions checked.
- [ ] Entity conversion trigger documented.
- [ ] No personal guarantees signed without review.

## 5. Digital identity setup

- [ ] Domain(s) purchased.
- [ ] DNS secured.
- [ ] Email provider configured.
- [ ] SPF configured.
- [ ] DKIM configured.
- [ ] DMARC configured.
- [ ] `hello@` created.
- [ ] `support@` created.
- [ ] `privacy@` created.
- [ ] `security@` created.
- [ ] Password manager vault created.
- [ ] MFA enabled.
- [ ] Recovery codes stored.
- [ ] Backup owner process documented.

## 6. Website before public launch

- [ ] Home page.
- [ ] About/founder page.
- [ ] Product/service page.
- [ ] Official accounts/links page.
- [ ] Contact page.
- [ ] Support page.
- [ ] Privacy page.
- [ ] Terms page if paid or accounts exist.
- [ ] Refund/cancellation page if paid.
- [ ] Trust/claims caveat page.
- [ ] Accessibility statement only if reviewed and accurate.
- [ ] Cookie/analytics notice if needed.

## 7. Registers to instantiate

Create operational copies from templates:

- [ ] account register;
- [ ] access review log;
- [ ] processing register;
- [ ] AI tools register;
- [ ] vendor/processor register;
- [ ] claims register;
- [ ] proof/testimonial permission register;
- [ ] incident/datalek register;
- [ ] payment/tax architecture decision log;
- [ ] product risk register.

## 8. Revenue readiness by lane

### App-store revenue

- [ ] App Store / Play Console accounts ready.
- [ ] Store agreements/tax/banking complete.
- [ ] RevenueCat configured if subscriptions.
- [ ] Privacy/data-safety labels drafted.
- [ ] Support/terms/privacy URLs live.

### Web micro-SaaS revenue

- [ ] MoR vs Stripe decision memo completed.
- [ ] Provider eligibility confirmed.
- [ ] Tax/VAT/OSS implications reviewed.
- [ ] Terms/refund/cancel pages drafted.
- [ ] Test transaction completed before public launch.

### Consulting revenue

- [ ] Service offer scoped.
- [ ] SOW/MSA reviewed.
- [ ] Liability cap reviewed.
- [ ] Insurance reviewed.
- [ ] Invoice/VAT treatment confirmed.
- [ ] Proof-permission clause included.

## 9. Monthly admin cadence

- [ ] reconcile accounting;
- [ ] review invoices/payouts;
- [ ] review tax/VAT thresholds;
- [ ] review access register;
- [ ] review processing register;
- [ ] review claims register;
- [ ] review incidents/support issues;
- [ ] update proof log;
- [ ] update decision log.
